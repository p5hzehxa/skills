# WorkOS SSO API Reference

## When to Use

Use this skill when building direct SSO integrations via REST API (instead of AuthKit). This is for scenarios requiring custom authentication flows, headless implementations, or integration with non-standard frontend frameworks. If you're using React, Next.js, or vanilla JavaScript in a standard web app, use the AuthKit skills instead.

## Key Vocabulary

- **Connection** `conn_` — links an Organization to an Identity Provider (e.g., Okta, Azure AD)
- **Organization** `org_` — the tenant/workspace entity that owns SSO connections
- **Profile** — the authenticated user object returned after SSO login
- **Authorization URL** — the IdP redirect URL generated by `/sso/get-authorization-url`
- **Redirect URI** — the callback URL where WorkOS sends users after IdP authentication
- **State parameter** — CSRF protection token passed through the SSO flow
- **`WORKOS_API_KEY`** — server-side authentication credential
- **`WORKOS_CLIENT_ID`** — public identifier for your WorkOS application
